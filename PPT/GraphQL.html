  • GraphQL 简介

        GraphQL 是一个由Facebook提出的 应用层查询语言.

         GraphQL 的客户端可以很灵活地在服务器能力范围内进行各种查询的组合，这种能力对向后兼容和版本控制很有好处。REST API 在进化的过程中往往要随着 API 版本而变迁， GraphQL API 基本没这个必要。对于同一 API，服务器只需要添加新的 field，新的客户端查询时使用新的 field 即可，不会影响老客户端。这是一种很优雅的进化方案。

  • GraphQL 图示
  • GraphQL 两种操作
  • query：仅获取数据（fetch）的只读请求

 

  • mutation：获取数据后还有写操作的请求
  • GraphQL 字段（Field）格式

 

                         alias:name(argument:value)  

 
  • alias 是字段的别名，即结果中显示的字段名称。
  • name 为字段名称，对应 schema 中定义的 fields 字段名。
  • argument 为参数名称，对应 schema 中定义的 fields 字段的参数名称。
  • value 为参数值，值的类型对应标量类型的值。

          例：        xxx/?query={user{userName:name(id:"1")}}

在查询里面，可以用不同的条件多次查询同一个字段，然后使用别名区分。

  • GraphQL:mutation
{
    query: `
    mutation M($name: String!) {
      createUser(name: $name) {
        name
        friends {
          name
        }
      }
    }
    `,
    params: {
      name: 'Wedge'
    }
  }
  • GraphQL:mutation
{
    query: `
    mutation M($userId: String! $name: String!) {
      updateUser(id: $userId name: $name) {
        name
        friends {
          name
        }
      }
    }
    `,
    params: {
      userId: userId,
      name: name
    }
  }
  • GraphQL:mutation
{
    query: `
    mutation M($userId: String!) {
      deleteUser(id: $userId) {
        name
        friends {
          name
        }
      }
    }
    `,
    params: {
      userId: deleteId
    }
  }
  • GraphQL 片段（Fragment）

         片段是 GraphQL 的主要组合数据结构，通过片段可以复用重复的字段，减少 query 中的重复内容。片段又分为 FragmentSpread 和 InlineFragment。

  • GraphQL 片段（Fragment）
query noFragments {
  user(id: 4) {
    friends(first: 10) {
      id
      name
      avatar(size: 50)
    }
    mutualFriends(first: 10) {
      id
      name
      avatar(size: 50)
    }
  }
}

query withFragments {
  user(id: 4) {
    friends(first: 10) {
      ...friendFields
    }
    mutualFriends(first: 10) {
      ...friendFields
    }
  }
}

fragment friendFields on User {
  id
  name
  avatar(size: 50)
}

=

使用片段时需要加上 ... 操作符表示展开片段内容。
